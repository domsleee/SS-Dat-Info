# UV Data Analysis: zeppelin.HX vs gameExtracted_zeppelin.ts

## Summary

**UV coordinates are NOT stored in the zeppelin.HX file.** They are generated procedurally by the game engine at runtime.

## Evidence

### 1. File Search Results

Searched the entire zeppelin.HX file (23,303 bytes) for the UV values found in `gameExtracted_zeppelin.ts`:

| UV Value | Found in HX? |
|----------|--------------|
| 0.999500 | ✗ NO |
| -0.999500 | ✗ NO |
| 0.116227 | ✗ NO |
| 0.000500 | ✗ NO |
| 0.918343 | ✗ NO |

**None of the UV values from the game-extracted data exist in the HX file.**

### 2. Mesh Structure Comparison

| Property | HX File | gameExtracted_zeppelin.ts | Notes |
|----------|---------|---------------------------|-------|
| Vertices | **199** | **269** | Game expands/duplicates vertices |
| Triangles | 324 | 324 | Same count ✓ |
| UV Data | **NONE** | **YES (269 pairs)** | Generated at runtime |

### 3. What's Actually in the HX File

The zeppelin.HX file contains only:

```
Offset 0xEB4: "mesh\0" marker
  ↓
Offset 0xEB9: Triangle indices (324 triangles × 12 bytes = 3,888 bytes)
  ↓
Offset 0x1DE9: Vertex positions (199 vertices × 12 bytes = 2,388 bytes)
  ↓
Offset 0x273D: Unknown data / padding (not UV coordinates)
```

### 4. How UVs are Generated

According to `HX_FILE_FORMAT.md`, the game engine uses **planar projection mapping**:

```c
// From sr.c line 64537: srModeler::planarMap()
U = ((vertex[axis_u] - bbox_min_u) * scale_u) + offset_u
V = ((-vertex[axis_v] - bbox_min_v) * scale_v) + offset_v
```

The UV data in `gameExtracted_zeppelin.ts` was captured from **game memory** using Cheat Engine **after** the engine generated the UVs at runtime.

## Conclusion

✅ **The zeppelin.HX file does NOT contain UV data**  
✅ **UV coordinates are procedurally generated by the game engine**  
✅ **The gameExtracted_zeppelin.ts UVs come from runtime memory, not the file**

This matches the reverse-engineering documentation perfectly:
> "UV coordinates are **NOT** stored in the HX file format. The SurRender engine generates UV coordinates procedurally at runtime using **planar projection mapping**."
